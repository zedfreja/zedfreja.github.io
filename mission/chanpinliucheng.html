产品流程

设计
  1. 产品方案确定后，将完成的 RFC 提给设计师开始 app 和 web 的设计（视情况可提前）。
  2. 设计师交付后，带上 RFC 和设计图，提需求给两个客户端负责人（需求冻结前），详细描述产品方案和细节，告知希望在哪一版本发布。
两个客户端（iOS/android）
  1. 两端负责人确认需求后，给出开发周期和开发人员。
  2. 有开发周期和开发人员后，邀请两个客户端开发和后端接口开发人员，一起确认联调时间，评估交付日期是否可以赶上灰度测试和发版时间。
web 前端交互开发
  1. 设计师交付后，带上 RFC 和设计图，提需求给前端开发负责人，详细描述产品方案和细节，确认开发周期和开发人员。
  2. 提需求给前端时，有可能要叫上各客户端开发、后端开发，一起确认前端开发细节和逻辑实现。
后端开发
  1. 确认前端开发细节和排期后，与后端开发确认排期。
  2. 后端如能提前知道前端接口规则，可先行。
埋点
  1. 找数据分析师进行需求 review
  2. 分析师完成埋点文档，把文档提给工程师
  3. 工程师在代码中添加埋点的代码
  4. 分析师测试埋点和报表
测试(QA)验收流程
  1. 将产品 RFC 提给 QA 工程师，详细描述产品逻辑细节、三端交互细节。
  2. 客户端灰度测试前 x 天，找对接 QA 工程师进行测试验收。
  3. 测试通过后，QA 环节完成验收。
